## 四维分析

### 1. 洞察力 — 发现了什么问题？

- **乘客**：打出租车难、不知道车在哪、安全感低
- **司机**：空驶率高（约50%），收入不稳定
- **城市**：私家车利用率低（约4%使用时间），停车资源浪费

### 2. 思维力 — 如何解决问题？

**设计思维应用：**

| 阶段 | 滴滴的做法 |
|------|----------|
| 同理心 | 实地观察打车场景，访谈司机和乘客 |
| 定义问题 | 供需匹配低效 + 信息不透明 |
| 创意发散 | 出租车叫车 → 快车（私家车共享） |
| 原型验证 | 北京出租车市场快速验证 |
| 测试迭代 | 动态定价（潮汐定价）平衡供需 |

**双边平台设计：**
- 乘客端：实时定位、预估价格、路线透明
- 司机端：接单流程优化、导航集成、收入可视化
- 平台：动态定价算法 → 高峰时期价格激励司机出行

### 3. 学习力 — 跨界融合了什么？

- 借鉴 Uber 的双边平台模式 → 本土化改造（出租车先切入）
- 运筹优化：借鉴供应链调度算法 → 骑手/司机派单
- 保险：与保险公司合作 → 解决安全顾虑（信任问题）

### 4. 表现力 — 可视化草图

```
[ 乘客发起叫车 ]                [ 司机空闲在线 ]
        ↓                              ↓
        └──────── [ 匹配算法 ] ────────┘
                        ↓
              距离/ETA/动态定价
                        ↓
              [ 接单 → 导航 → 完成 ]
                        ↓
              [ 双向评价 + 结算 ]
```

---

## 答题关键词

- **双边市场**：需要同时激活司机端和乘客端（冷启动难题）
- **网络效应**：司机越多→等待越短→乘客越多→司机越多（正向飞轮）
- **动态定价**：高峰期溢价，均衡供需（价格机制）
- **信任设计**：实名制 + 行程分享 + 录音功能（安全感构建）
