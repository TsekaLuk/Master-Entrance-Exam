# Harvey 产品案例分析

> 843 支撑案例 | 核心角度：**垂直行业 AI 的落地路径——专业壁垒与信任设计**

---

## 基本信息

| 项目 | 内容 |
|------|------|
| 产品定位 | 面向律师事务所的 AI 法律助手 |
| 所属公司 | Harvey AI |
| 创立时间 | 2022 年 |
| 融资 | 1 亿美元 A 轮（2023），估值 7.15 亿美元 |
| 投资方 | OpenAI Startup Fund（战略投资） |
| 用户群 | 顶级律所（Allen & Overy、普华永道法律等）|
| 商业模式 | 企业订阅（按用户/按使用量）|
| 核心技术 | GPT-4 基础 + 法律专有数据微调 |

---

## 四维速记

### 洞察力
- **问题**：法律工作 70% 是文档密集型重复劳动（合同审查、法规检索、备忘录起草），但每小时收费 $500-1000
- **关键洞察**：法律行业有**极高的专业壁垒**，通用 AI 不够用（错误引用法规后果严重）；但大量工作**可以被 AI 加速**，律师的价值在于判断，不在于检索
- **准入门槛**：律所对数据安全要求极高，反而形成竞争护城河（一旦进入不易被替换）

### 思维力
**核心设计决策：专业 + 信任**
```
产品设计三原则：
  1. 引用来源可追溯（每个AI回答注明出处法规/案例）
  2. 律师人工确认（AI建议，律师决策，不完全替代）
  3. 数据隔离（每家律所的数据私有，不交叉训练）

与通用AI的关键差异：
  通用 AI：知识截止日期 + 无法律专业语境
  Harvey：实时法规库 + 法律微调 + 律所私有文档 RAG
```

### 学习力
| 借鉴来源 | 融合方式 |
|---------|---------|
| Thomson Reuters Westlaw | 法律数据库的用户习惯，Harvey 更新交互层 |
| Salesforce（CRM）| 律所 CRM 集成，融入已有工作流 |
| OpenAI（GPT-4）| 底层模型 + OpenAI 战略投资（背书效应）|

### 表现力
```
垂直 AI 落地公式：
  通用大模型
    + 行业专有数据（法律法规/判例）
    + 信任设计（引用可追溯/人工审核）
    + 客户数据私有化
  = 专业壁垒 × 切换成本极高
```

## 843 答题关键词

| 维度 | 关键词 |
|------|--------|
| 垂直化 | 专注单一行业，深度微调超越通用AI，形成专业护城河 |
| 信任设计 | 引用来源可追溯、人工确认机制，解决高风险行业的AI信任问题 |
| 企业SaaS路径 | 从顶级律所切入 → 背书效应 → 扩散至中型律所 |
| 可复制范式 | Harvey模式适用于医疗/财务/会计等所有专业服务行业 |
