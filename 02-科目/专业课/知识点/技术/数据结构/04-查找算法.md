# 查找算法 ⭐⭐⭐

## 三种核心查找算法对比

| 算法 | 适用结构 | 前提条件 | 时间复杂度 | 空间复杂度 |
|------|---------|---------|-----------|-----------|
| 顺序查找 | 线性表（数组/链表） | 无 | O(n) | O(1) |
| 折半查找（二分） | 有序顺序表 | **必须有序** | O(log n) | O(1) |
| 散列查找（哈希） | 散列表 | 需构建哈希函数 | O(1)（理想）| O(n) |

---

## 1. 顺序查找

**思路：** 从头到尾逐个比较，找到返回下标，找不到返回-1

```
arr = [3, 1, 4, 1, 5, 9, 2, 6]
查找 5：
  比较 3 → 1 → 4 → 1 → 5 ✓  第5次找到
```

**平均查找长度（ASL）：**
$$ASL = \frac{n+1}{2}$$

适用场景：数据量小、无序、链式存储

---

## 2. 折半查找（二分查找）

**思路：** 每次取中间元素与目标比较，缩小一半范围

```
有序数组：[1, 3, 5, 7, 9, 11, 13]
查找 7：
  low=0, high=6, mid=3 → arr[3]=7 ✓ 找到
```

**平均查找长度（ASL）：**
$$ASL \approx \log_2(n+1) - 1$$

**判定树**（二分查找过程 = 二叉排序树）：
- 比较次数 = 判定树高度 = ⌈log₂(n+1)⌉

适用场景：**有序顺序表**（不适合链表，因为不能随机访问）

---

## 3. 散列查找（哈希查找）

### 哈希函数设计

| 方法 | 公式 | 说明 |
|------|------|------|
| 除留余数法 | H(key) = key mod p | p 取质数，最常用 |
| 直接定址法 | H(key) = a·key + b | 无冲突，需连续地址空间 |
| 数字分析法 | 取关键字中分布均匀的几位 | 适合关键字位数多 |

### 冲突解决

| 方法 | 说明 | 优缺点 |
|------|------|--------|
| 开放定址法（线性探测）| 冲突时找下一个空位 | 易产生聚集 |
| 链地址法（拉链法）| 冲突元素用链表串起来 | 无聚集，删除方便 |
| 再哈希法 | 冲突时换哈希函数 | 计算代价高 |

### 平均查找长度

装填因子 α = 表中记录数 n / 散列表长度 m

| 装填因子 α | 线性探测 ASL（查找成功）|
|-----------|----------------------|
| 0.5 | ≈ 1.5 |
| 0.75 | ≈ 2.5 |
| 0.9 | ≈ 5.5 |

> **结论：** α 越小，冲突越少，查找越快；一般控制 α ≤ 0.75

## 易混淆点

| 问题 | 答案 |
|------|------|
| 二分查找能用链表吗？ | **不能**，链表不支持随机访问 |
| 哈希查找最坏是多少？ | O(n)（全部冲突退化为顺序查找）|
| 折半查找适合动态插入吗？ | **不适合**，维护有序代价高 |
| 散列表最好查找是多少？ | O(1)，无冲突时 |
