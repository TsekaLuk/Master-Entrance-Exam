# Retool 产品案例分析

> 843 支撑案例 | 核心角度：**低代码内部工具——B2B 生产力工具的垂直切入**

---

## 基本信息

| 项目 | 内容 |
|------|------|
| 产品定位 | 面向工程师的低代码内部工具构建平台 |
| 创立时间 | 2017 年（Y Combinator）|
| 融资 | 1.45 亿美元 C 轮（2022），估值 33.85 亿美元 |
| 用户群 | 工程师/产品经理，构建内部管理后台 |
| 商业模式 | 按用户数订阅（$10-50/用户/月）+ 企业版 |
| 核心用途 | 管理后台、数据看板、客服工具、运营面板 |

---

## 四维速记

### 洞察力
- **问题**：每个公司都需要"内部工具"（订单管理/用户查询/数据看板），但从头开发耗时长、工程师不情愿
- **关键洞察**：内部工具有高度相似性（表格+按钮+表单+图表），可以提取为可复用的模块；工程师是真正的决策者，产品要让工程师爽

### 思维力
**核心设计：面向工程师的低代码**
```
不同于 Bubble（面向非技术人员）：
  Retool 面向工程师：
    ├── 支持写 SQL/JavaScript（不完全拖拉拽）
    ├── 直连数据库/API（Postgres/REST/GraphQL）
    └── 版本控制（Git 集成）

核心价值：
  从零开发内部工具需要 2 周
  Retool 搭建只需 2 小时
```

### 843 答题关键词

| 维度 | 关键词 |
|------|--------|
| 垂直切入 | 专注"内部工具"这一细分场景，而非通用低代码 |
| 工程师友好 | 保留代码能力（SQL/JS），不强迫纯拖拽 |
| 企业 B2B | 从初创到Fortune 500，内部工具需求普遍存在 |
