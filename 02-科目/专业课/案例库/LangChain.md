# AI Agent 框架案例分析
## LangChain · LangGraph · CrewAI

> 843 支撑案例 | 核心角度：**从链式调用到多智能体协作——AI 应用的编排层**

---

## LangChain

| 项目 | 内容 |
|------|------|
| 定位 | LLM 应用开发框架，链式调用模式 |
| 创立 | 2022 年（Harrison Chase）|
| 融资 | 2500 万美元 A 轮（2023），估值 2 亿美元 |
| 核心价值 | 将 LLM、工具、记忆、数据源 "链" 接在一起 |

**核心概念：**
```
LangChain 的 Chain 模式：
  用户输入
    → 检索相关文档（RAG）
    → 提示词模板
    → LLM 调用
    → 输出解析
    → 工具调用（搜索/计算器）
    → 最终回答

价值：封装重复模式，开发者 3 行代码实现 RAG
局限：链式结构灵活性差，复杂任务需要 LangGraph
```

---

## LangGraph

| 项目 | 内容 |
|------|------|
| 定位 | 有状态多智能体编排框架（LangChain 出品）|
| 核心创新 | 图结构（而非链结构），支持循环和条件分支 |

**核心概念：**
```
LangGraph 解决 LangChain 的局限：

LangChain（链）：A → B → C → 结束
LangGraph（图）：A → B → 判断 → C 或 D → 循环或结束

适用场景：
  ├── 需要"思考-行动-观察-再思考"的 ReAct 模式
  ├── 多个 Agent 协作完成任务
  └── 需要持久化状态的长任务（跨多次对话）
```

---

## CrewAI

| 项目 | 内容 |
|------|------|
| 定位 | 角色化多 Agent 协作框架 |
| 核心创新 | 为每个 Agent 定义"角色"和"目标"，像团队协作 |

**核心概念：**
```
CrewAI 的设计哲学：
  不是写代码，而是"招募团队"

  创建一个研究小组：
    研究员 Agent：搜索最新信息
    分析师 Agent：整理关键洞察
    写作者 Agent：生成报告

  Agent 之间可以传递任务，就像真实工作团队
```

---

## 843 答题关键词（三框架统一）

| 框架 | 843 考试角度 |
|------|-------------|
| LangChain | AI应用开发的"积木"——封装LLM能力，降低应用开发门槛 |
| LangGraph | 多智能体编排——有状态、可循环、支持复杂任务 |
| CrewAI | 角色化AI团队——类比人类组织设计迁移到AI协作 |

**整体价值（843 答题用）：**
> AI 框架层是 AI 生态的"中间件"——连接基础模型（OpenAI/Claude）和具体应用（Cursor/Harvey），是 AI 应用开发的操作系统
